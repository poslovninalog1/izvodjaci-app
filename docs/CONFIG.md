# Root configuration reference

Next.js expects these files at the **repository root**. Do not move them.

---

## package.json / package-lock.json

- **package.json** — Project name, scripts, dependencies. Defines `dev`, `build`, `start`, `lint`, Supabase scripts, and **dev:no-turbo** (runs dev with Webpack instead of Turbopack; use if you see Tailwind/PostCSS/lightningcss errors).
- **package-lock.json** — Locked dependency versions. Commit it for reproducible installs.

---

## next.config.ts

- Next.js framework config (TypeScript).
- Used for: `reactStrictMode`, future overrides (images, redirects, env, etc.).
- Build and dev server read this from root.

---

## tsconfig.json

- TypeScript compiler options for the project.
- **paths**: `@/*` → `./*` (repo root), so imports like `@/app/...` and `@/src/lib/...` resolve correctly.
- **include**: App, `next-env.d.ts`, Next.js generated types under `.next/types`.
- **exclude**: `node_modules`. Do not move; Next.js and `tsc` expect it at root.

---

## postcss.config.mjs

- PostCSS config for CSS processing (Tailwind, etc.).
- **This project uses Tailwind v3**: plugins are `tailwindcss` and `autoprefixer` (function form: `tailwindcss()`).
- Next.js runs PostCSS on `app/globals.css` and any imported CSS. File must stay at root.

---

## Tailwind setup

- **Version**: Tailwind v3 (see `package.json`: `tailwindcss` ^3).
- **Config**: `tailwind.config.ts` at root defines `content` paths; `postcss.config.mjs` uses `tailwindcss: {}` and `autoprefixer: {}`.
- **Entry CSS**: `app/globals.css` must start with:
  - `@tailwind base;`
  - `@tailwind components;`
  - `@tailwind utilities;`
- **If you see** “Cannot find module 'unknown'” (lightningcss / @tailwindcss/node) with Turbopack, run **`npm run dev:no-turbo`** to use Webpack instead.

---

## eslint.config.mjs

- ESLint flat config. Uses `eslint-config-next` (core-web-vitals + TypeScript).
- **globalIgnores**: `.next/**`, `out/**`, `build/**`, `next-env.d.ts`.
- Must stay at root so ESLint and editor integrations find it.

---

## next-env.d.ts

- Auto-generated by Next.js (TypeScript declarations). Listed in `.gitignore` or committed depending on setup. Do not move.

---

## .gitignore

- Tells Git to ignore build/cache and secrets. Key entries:
  - **Build/cache**: `.next/`, `node_modules/`, `*.tsbuildinfo`, `/out/`, `/build/`
  - **Env**: `.env*`, with `!.env.local.example` so the example is committed
  - **AI/editor**: `.aider*`, `.cursor*`
  - **Logs**: `logs/`
  - **Supabase CLI**: `supabase/.temp/`, `supabase/.branches/`

---

## Env files

- **.env.local** — Local env (not committed). Holds `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY` for the Next.js app.
- **.env.local.example** — Committed template; copy to `.env.local` and fill values.
- **.env**, **.env.development**, **.env.production** — Optional; all matched by `.env*` in `.gitignore`. Never commit secrets.

All env files are read by Next.js from the **root**; do not move them.
